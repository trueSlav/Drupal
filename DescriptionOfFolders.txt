vendor
settings.php
Компосер работает на основе двух файлов:
1) composer.json - содержит всю информацию, какие пакеты требуются, настройки, скрипты и.т.д.
   Его можно править, менять структуру.
2) composer.lock - содержит актуальную информацию о текущем проекте. В него записываются все пакеты,
   которые установлены и прямые пути на их загрузку.

    web
    (в web)
    modules - для установки модулей.
    profiles - для установки профилей  установки.
    sites - содержит настройки и файлы конфигурации, относящиеся к сайту.
    themes - для установки тем.
            (в  web/core )
            assets/scaffold - файлы, содержащиеся внутри core, но установлены за пределами основного каталога.
            config - конфигурация.
            includes - библиотеки общих функций.
            lib - предназначен для классов, представляемых DCore, которые являются оригинальными.
            misk - JS сценарии, изображения, которые входят в базовый пакет Drupal.
            modules - содержит модули ядра.
            profiles - содержит различные профили установки.
            script - сценарии, которые используются для проверки синтаксиса, очистки кода, запуска из ком.строки.
            test - тест различных php файлов и не только. Тест конфигурации.
            themes - содержит темы ядра.
    vendor
    (в vendor)
    asm89 - библиотека и промежуточное ПО, обеспечивающее совместное использование ресурсов из разных источников.
